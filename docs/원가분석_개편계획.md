# 원가분석 개편 계획 v3 — 3년 재무제표 · 원가회계 방법론 · **물리 BOM(수량×단가) + 노무비 검증**

> **v3 변경 핵심 (→ §11 물리 BOM · §12 노무비)**
> 0. **노무비(사람값) 축 추가**: 재료비만 세면 마진이 과대평가된다(치킨표의 구멍).
>    DART 「직원 등의 현황」의 **직원수·1인평균급여 실측** + **취업사이트 평균연봉 교차검증** + **하청(소속 외 근로자)** 까지 → 제조원가 3요소(재료·노무·경비)로 표를 닫는다.
> 1. **물리 BOM 원가분석 도입**: "치킨 1마리 = 닭 6,600 + 파우더 800 + 기름 860 + 박스 150 …" 처럼
>    **투입물 수량 × 조달단가**를 더해 원가를 **아래에서부터 다시 쌓는다**(현재는 원가율 % 배분 = 위에서 아래).
> 2. **동기: 재무제표는 편법으로 조작 가능하지만 물리량(투입 톤수·생산 대수·원단위)은 조작이 훨씬 어렵다.**
>    → 물리 BOM으로 만든 **이론 원가**와 공시 매출원가를 대조해 **괴리 = 회계 이상 신호**로 쓴다.
> 3. 세분화 축을 **회사 → 품목 → SKU → 투입물(수량·단위·단가·출처)** 4단으로 확장.

> **v2 변경 핵심**
> 1. **분석 기간을 최소 3개년**으로 확장(재무제표·원가요소·차이 모두 3년 추세로).
> 2. 표준원가계산·종합원가계산·결합원가계산·공손/부산물 등 **원가회계 방법론을 분석 프레임워크로 채택**.
> 3. 단, 이 방법들은 원래 **회사 내부 데이터**(표준소요량·완성도·재공품수량·검사시점 등)를 전제한다.
>    우리는 **외부 공개데이터(DART)만** 있으므로, 각 개념을 **"공개데이터로 계산가능 / 추정·근사 / 내부데이터 필요→해석·교육용"** 으로 구분해 현실적으로 적용한다.

> 기존 목표(유지): 회사당 대표 물건 1개가 아니라, **회사 → 품목 전체 원가·영익 → 원재료 시세 → 마진 정합성 → 재무제표 근거** 3단 드릴다운.

## 1. 3개년 분석으로 확장 (v2 신규 골격)

모든 지표를 **최근 3개 사업연도(FY-2 · FY-1 · FY)** 로 가져와 **추세**로 본다.

- 🟢 **3년 손익 추세**: 매출·매출원가·판관비·영업이익 및 각 비율(원가율·판관비율·영익률)의 3년 변화.
- 🟢 **3년 원가율 궤적**: 매출원가율이 오르는지/내리는지 → 원가 압박 방향.
- 🟡 **3년 원가차이 추세**: 아래 §2 표준원가(변형) 차이(가격/능률)를 3년으로 → 원자재발 압박인지 효율 문제인지 구조적 판별.
- 🟢 **3년 원재료 시세 vs 원가율 동행성**: 원자재가 올랐는데 원가율이 안 올랐다 = 전가력(pricing power) 방증.

**데이터**: `store.dart_financials(ticker)` 는 이미 **연도별(year)** 로 계정을 저장 → 최근 3년 슬라이스만 하면 됨(`_income_ratios` 를 3년 반환형으로 확장).

## 2. 원가회계 방법론 프레임워크 (사용자 요구 반영)

각 방법을 **① 원론 요지 → ② 우리 앱에서의 적용 → ③ 실현가능성** 순으로.

### 2.1 표준원가계산 · 원가차이 분석 → **"기준원가 vs 실제원가" 차이분해** 🟡

**원론**: 직접재료비·직접노무비·제조간접비의 표준원가를 사전 설정하고 실제원가와 비교, 차이를 가격차이/능률차이로 분해(F/U).
```
        AQ×AP ── (가격차이) ── AQ×SP ── (능률차이) ── SQ×SP
```
**우리 적용(외부데이터 버전)**: 내부 표준(SQ·SP)은 공시되지 않으므로, **"기준원가(standard proxy)"** 를 두 소스로 세운다.
- **기준단가(SP proxy)** = 원자재 국제시세(`commodities`) 기준 원가.
- **실제원가(actual)** = DART 매출원가(실측).
- **차이 분해(근사)**:
  - 🟡 **가격차이 ≈ 원자재 시세 변동분** = 원재료비 × (원자재 1년/3년 등락률). "원가가 오른 게 재료값 때문인가?"
  - 🟡 **능률·기타차이 ≈ 잔차** = (실제 매출원가 − 기준원가) − 가격차이. 판가전가·믹스·효율·환율 등 나머지.
  - 🟢 **F/U 판정**: 실제 < 기준 → 유리(F), 실제 > 기준 → 불리(U). 3년 추세로.
- ⚪ **직접노무·제조간접의 표준임률/표준시간 분해는 불가**(공시 없음) → "판관비·가공비 3년 추세"로 대체 제시하고 한계 명시.

> 이는 기존 **마진 정합성(Phase 4)** 을 표준원가 언어로 정식화하고 **가격차이/능률차이 2요소로 분해 + 3년화**한 것.

**차이의 회계처리(교육 카드, ⚪)**: 매출원가조정법/기타손익법(손익법), 총원가기준법/원가요소기준법(배분법)은
외부에서 재현 불가 → **"표준원가제 채택 기업은 이런 방식으로 실제원가 전환한다"** 설명 툴팁으로만.

### 2.2 종합원가계산 → **생산유형별 원가모델 분류 체계** 🟢(분류) / ⚪(환산량)

**원론**: 동일공정 연속 대량생산에 적용. 완성품환산량 = 재공품수량 × 완성도, 선입선출법/평균법으로 기말재공품 평가.

**우리 적용**: 완성품환산량·완성도·재공품 평가는 **내부데이터 필요 ⚪** → 수치 계산 불가.
대신 **생산유형 분류를 원가모델 아키타입 선택에 사용** 🟢:

| 종합원가 유형 | 적용 업종(예) | 우리 원가모델 아키타입 |
|---|---|---|
| 단일(단순) | 제빙·제염·제와 | 단일품목 원가율 |
| 공정별 | 화학·제지·제당 | 공정투입물(나프타 등) 스프레드형 |
| 조별(반별) | 식료품·제과·통조림·직물 | 품목(조)별 원가 분리 |
| 등급별 | 제분·제화·양조·화학약품 | 등급별 배분(§2.4) |
| 결합(연산품) | 정유·정육 | 결합원가 배분(§2.4) |

→ 회사를 **업종→생산유형→아키타입**으로 자동 태깅해, 어떤 원가모델을 쓸지 결정.
⚪ 재공품/완성도 항목은 **"이 업종은 재공품 비중이 커서 원가배분법(선입선출 vs 평균)에 따라 원가가 달라진다"** 는 해석 카드로.

### 2.3 기말재공품 평가(선입선출법 vs 평균법) → **해석·교육 레이어** ⚪

**원론**: 완성품제조원가 = 기초재공품 + 당기총제조비용 − 기말재공품. 분자/분모가 선입선출·평균법에 따라 다름.

**우리 적용**: 재공품수량·완성도 미공시 ⚪ → 계산 불가.
- 🟢 **대체 관측**: DART 재무상태표의 **재고자산 주석(원재료·재공품·제품)** 3년 추이는 공시됨 → 재공품 비중·재고 증감을 보여주고, 재고평가법(선입선출/평균/총평균)이 주석에 명시되면 표기.
- ⚪ 완성품환산량 예제·공식은 **원가교육 툴팁**으로(요구한 예제 포함).

### 2.4 결합원가계산(연산품·등급별) → **다품목 회사 원가배분** 🟢/🟡

**원론**: 한 공정에서 2종↑ 제품(정유→휘발유·등유·경유; 콩→두부·비지). 분리점 이전 결합원가를 배분:
상대적 판매가치법 / 순실현가치법(NRV) / 물량기준법.

**우리 적용(가장 실전적으로 유효)** 🟡:
- 정유·석유화학·정육·유가공·제분 등 **연산·등급 업종**은 회사 매출원가(결합원가 proxy)를 **제품별 매출가치 비중으로 배분** → 품목별 원가·마진 추정.
  - 🟡 **상대판매가치법**: 결합원가 × (품목 매출액 ÷ 총매출액). 부문별 매출(사업보고서)로 계산가능.
  - 🟡 **순실현가치법**: (품목 매출 − 추가가공비·판매비) 기준. 추가가공비는 근사.
  - 🟢 **물량기준법**: 생산량·중량 공시 시.
- **주산품/부산품 구분**: 매출비중 임계값으로 자동 태깅(예: <5% = 부산품 후보). ⚪ 절대기준 없음 → 임의·시장가치 기준임을 명시.

### 2.5 공손품·작업폐물·부산물 → **원가율 이상치 해석 플래그** 🟡

**원론**: 정상공손=제품원가 포함, 비정상공손=영업외비용. 작업폐물=재료비/제조원가 차감·잡이익. 부산물=주산품 대비 소액.

**우리 적용**:
- 🟡 **비정상공손·재고평가손실 탐지**: 3년 중 매출원가율이 튀거나 **재고자산평가손실·기타비용**이 급증한 해 → "비정상 원가(공손·감모·평가손) 발생 가능" 플래그. 회계 착시 탐지(`earnings_quality.py`)와 연계.
- 🟡 **부산물 수익**: 잡이익·기타수익 3년 추이로 부산물/스크랩 매각 규모 근사.
- ⚪ 정상/비정상 공손 정확 구분·검사시점 배분은 내부데이터 필요 → 해석 카드.

> 아래는 DART 캐시에서 뽑은 **실측 3개년**으로, 화면에 그대로 나올 수를 손계산한 것.
> (수치 출처: `store.dart_financials`, 단위 억원)

### (A) 농심 004370 — 표준(기준)원가 vs 실제 차이분해 [조별 종합원가]

**1단계. 3개년 손익 실측** 🟢
| | FY2023 | FY2024 | FY2025 |
|---|--:|--:|--:|
| 매출액 | 34,106 | 34,387 | 35,143 |
| 매출원가 | 23,753 | 24,732 | 24,954 |
| **매출원가율** | **69.6%** | **71.9%** | **71.0%** |
| 영업이익 | 2,121 | 1,631 | 1,839 |
| **영업이익률** | **6.2%** | **4.7%** | **5.2%** |

→ 관찰: 원가율이 **69.6% → 71.0%(+1.4%p)** 오르며 영익률이 **6.2% → 5.2%(−1.0%p)** 훼손. **"왜?"** 를 차이분해로 규명.

**2단계. 매출원가를 원가요소로 분해** 🟡 (FY2025 기준, `material_ratio_of_cogs=0.80`, 신라면 mix)
```
매출원가 24,954억
├ 원재료비  = 24,954 × 0.80 = 19,963억
│   ├ 소맥(밀) 24% = 4,791억
│   ├ 팜유    24% = 4,791억
│   ├ 봉지(BOPP) 27% = 5,390억
│   ├ 스프·건더기 16% = 3,194억
│   └ 기타     9% = 1,797억
└ 가공비(노무·감가·에너지) = 24,954 × 0.20 = 4,991억   ← ⚪ 표준임률/시간 분해는 미공시
```

**3단계. 원가차이 분해 (기준원가 vs 실제)** 🟡 — 두 시계열로 본다.

*장기(FY23→25):* 원가율 **+1.4%p 상승**(장기 원가 압박).
*최근 1년(FY24→25):* 원가율 **−0.9%p 개선**(단기 방어 성공) — 이 1년을 실제 시세로 분해:

**가격차이(원자재발, 최근 1년 실측 등락률)** = Σ(원재료비ᵢ × chg_1yᵢ)
| 원재료 | FY25 원재료비 | 최근1년 등락 | 가격차이(억) | F/U |
|---|--:|--:|--:|:--:|
| 소맥(밀) | 4,791 | −8% | **−383** | F |
| 팜유 | 4,791 | +9.8% | **+470** | U |
| 봉지(BOPP) | 5,390 | +15% | **+809** | U |
| 스프·건더기·기타 | 4,991 | 시세 미연동 | 0 | — |
| **가격차이 합** | | | **+895** | **U 불리** |

```
가격차이 +895억  →  원가율에 준 압력 = +895 / 35,143 = +2.5%p (U, 원자재가 올렸어야 할 몫)
실제 원가율 변화(FY24→25)            = 71.9% − 71.0% ... = −0.9%p (F, 오히려 개선)
능률·기타차이(잔차) = 실제(−0.9%p) − 가격(+2.5%p) = −3.4%p (F, 유리) ≈ −1,195억 상당
```
| 차이 항목 | %p | 금액(억) | F/U | 의미 |
|---|--:|--:|:--:|---|
| 가격차이 (원자재발) | +2.5%p | +895 | **U** | 팜유·봉지 상승이 원가 압박 |
| 능률·기타차이 (잔차) | −3.4%p | ≈−1,195 | **F** | 판가 인상·믹스·효율로 초과 방어 |
| **실제 원가율 변화(1년)** | **−0.9%p** | ≈−316 | **F** | 순개선 |

> 화면 결론 문구(자동): *"농심: 최근 1년 원자재는 원가율을 **+2.5%p 밀어올렸(U)**지만, 판가·믹스로 **−3.4%p 방어(F)** → 원가율 오히려 −0.9%p 개선. **전가력 강함.** 단, 장기(3년)로는 +1.4%p 눌린 상태."*
> ⚪ 노무·제조간접의 임률·시간차이는 미공시라 '가공비 3년 추세'로 대체.

### (B) S-Oil 010950 — 결합원가(연산품) 배분 [정유]

**1단계. 3개년 손익 실측** 🟢 — 정유는 매출원가율이 **판가(정제마진)** 에 따라 크게 출렁:
| | FY2023 | FY2024 | FY2025 |
|---|--:|--:|--:|
| 매출액 | 357,267 | 366,370 | 342,470 |
| 매출원가율 | 94.1% | 96.7% | **97.0%** |
| 영업이익률 | 3.8% | 1.2% | **0.7%** |

**2단계. 결합원가 배분 (상대판매가치법)** 🟡 — 정유 매출원가(FY25 **332,037억**)를 제품 매출비중으로 배분:
```
결합원가 배분ᵢ = 332,037 × (제품ᵢ 매출액 ÷ 총매출액)
  예) 경유 30% → 99,611억 · 휘발유 20% → 66,407억 · 나프타/기타 50% → 166,019억
    (제품 매출비중은 사업보고서 매출실적표에서 파싱 🟢)
```
→ 제품별 원가·마진 추정 → **어느 유종이 마진을 갉아먹는지** 표시. 주산품(경유·휘발유)/부산품(아스팔트 등) 자동 태깅.

### (C) 공손·이상치 플래그 예시 🟡
- S-Oil FY23→24 원가율 **94.1%→96.7%(+2.6%p)** 급등 → "정제마진 축소/재고평가손 가능" 플래그.
- 3개년 중 원가율 σ가 큰 종목(정유·화학·조선) 자동 리스트업 → 원가 변동성 경보.

  더 얹지 않는다(수치를 지어내지 않는 §0 원칙). **순실현가치법은 데이터가 있을 때만 보조 표시** —
  품목별 추가가공비·판매비는 미공시(⚪)라 공개데이터로 실제 계산되는 유일한 변형인
  **부산물 순액차감법**만, 부산품(매출비중 5% 미만)이 식별될 때 산출하고 화면에서 토글로 비교.
  화면에 반드시 병기하는 성질: 상대판매가치법은 **정의상 품목별 매출총이익률이 모두 같게** 나온다
  (균등마진 가정) → 여기서 읽을 것은 마진 차이가 아니라 "어느 품목이 결합원가를 얼마나 짊어지나".
  실측 예: S-Oil 정유 76.3%/윤활 13.0%/석화 10.7% 배분, 하림은 부산물 순액차감 시 주산품 GM 15.4%→17.7%.

- ✅ **v3: 노무비 축 추가** — DART 직원현황 실측(L1) 우선, 취업사이트 평균연봉은 교차검증·하청단가용(L2), 하청 규모는 고용형태공시(L3)(§12).
- ⬜ 취업사이트 연봉 수집 방식: 수동 입력 vs 공개 수치 자동 취합(ToS·레이트리밋 검토 필요) — §10.1과 동일 원칙 적용.
- ⬜ 연근로시간 상수(식6 분모) 기본값: 2,000h vs 업종별 차등.
- ✅ **⚪ 교육 레이어 = 툴팁 + 접이식 카드** (별도 탭 X) — `costing_edu.py`, 구현 완료.

  종목당 `sleep`(기본 0.4~1.0s)으로 DART rate limit 준수, `data/company_costmodels.json`
  원자적 교체(tmp→replace), 결과가 **레벨1 목록의 추정값을 DART 실측으로 덮어씀**(목록 속도 유지).
  재시작 시 파일 날짜를 보고 그날 몫 중복 실행을 막고, 서버가 꺼져 있었으면 켜진 뒤 따라잡는다.
  상태: `GET /api/data/company-costmodel/batch`. `COSTMODEL_BATCH=false` 로 끔.

# 11. 물리 BOM 원가분석 (v3 신규) — "치킨 1마리" 방식을 대기업으로

> 요구: *"치킨 원가표처럼 섬세한 걸 대기업에도 할 수 있나? 더더욱 세분화돼야 한다.
> 왜냐면 주가·재무제표는 편법으로 조작할 수 있으니까."*
> → **가능하다. 단, 업종별로 가능한 깊이가 다르고, 공시 물량데이터로 BOM을 역산해야 한다.**

## 11.1 지금 방식의 한계 (왜 새 축이 필요한가)

| | 현재(v1·v2) | v3 물리 BOM |
|---|---|---|
| 방향 | **Top-down**: 소비자가 → 유통마진 → 공시 원가율(%) → 원재료 비중(%)로 쪼갬 | **Bottom-up**: 투입물 **수량 × 조달단가**를 더해 원가를 쌓음 |
| 재무제표 의존 | **절대적** (매출원가율이 틀리면 전부 틀림) | **독립적** (재무제표를 안 보고도 원가가 나옴) |
| 조작 취약성 | 재무제표를 분식하면 **그대로 따라 틀림** | 물량·단가는 별도 출처 → **분식 탐지에 사용 가능** |
| 산출 | "원가율 71%" | "1개당 재료비 9,015원, 마진 8,985원" ← 치킨표 |

핵심: 현재 `unit_economics.PRODUCTS.material_mix[].weight`는 **비중(%)** 이다.
치킨표의 본질은 비중이 아니라 **`qty × unit_price = 금액`** 이다. 이 필드가 없어서 세분화가 막혀 있다.

```
치킨표 1행의 정보량:  닭 | 1마리 | 6,600원/마리 | 6,600원 | "9호닭, 브랜드마다 차이"
현재 우리 1행:        밀가루 | weight 0.24 | (수량 없음) | (단가 없음)
                                    ↑ 여기 3칸이 비어 있다
```

## 11.2 대기업 적용 가능성 등급 — 업종별

BOM을 얼마나 깊게 쌓을 수 있는지는 **"제품 1단위의 물리적 정의가 가능한가"** 로 갈린다.

| 등급 | 조건 | 업종 예 | 도달 가능 깊이 |
|:--:|---|---|---|
| **A** 🟢 치킨표 수준 가능 | 제품이 **중량·개수로 표준화**, 원단위가 공학적으로 알려짐 | 철강·시멘트·정유·석유화학·제지·비료·전력·유리 | 투입물별 **톤/kWh × 단가** 실수치 |
| **B** 🟡 SKU 표 가능(추정) | 소비재 SKU, 성분표·중량 공개 + 원자재 시세 존재 | 음식료·주류·담배·화장품·생활용품 | SKU 1개당 g/ml × 시세 |
| **C** 🟡 부품·모듈 단위까지 | BOM은 비공개지만 **주요 모듈 단가가 시장에 존재** | 자동차·2차전지·가전·조선·기계 | 모듈 5~15개 (배터리셀·강판·엔진) |
| **D** ⚪ 물리 BOM 부적합 | 원가의 대부분이 **인건비·R&D·감가상각**, 물량 정의 불가 | 반도체·소프트웨어·게임·바이오신약·금융·플랫폼 | BOM 대신 **웨이퍼/가동률/인당매출** 대체지표 |

> 결론: **대기업일수록 오히려 A·C가 많다**(장치산업). 치킨은 B급 난이도인데,
> 포스코·S-Oil·롯데케미칼은 **A급 = 치킨보다 더 정밀하게 가능**하다. 원단위가 공학 상수에 가깝기 때문.
> 반면 삼성전자 반도체는 D → 억지로 BOM 만들지 말고 **대체지표로 전환**(§11.7).

## 11.3 공시로 BOM을 만드는 3개의 역산식 (핵심 방법론) 🟢/🟡

DART 사업보고서 「II. 사업의 내용 → 3. 원재료 및 생산설비」에는 **금액·수량·단가·생산실적**이 흩어져 있다.
이 셋을 물리면 BOM이 복원된다.

```
[식1] 조달단가   P_actual = 원재료 매입액 ÷ 매입수량            🟢 (수량 공시 시)
                 ※ 미공시면 「가격변동추이」표의 단가 직접 사용 🟢
[식2] 원단위     U = 원재료 투입수량 ÷ 제품 생산수량            🟢 (생산실적표 존재)
                 ※ = "제품 1톤 만드는 데 원재료 몇 톤"  ← 치킨의 "닭 1마리/치킨 1마리"
[식3] BOM 원가   C_unit = Σ (U_i × P_actual_i) + 가공비원단위    🟡
                 → 이게 곧 치킨표의 "재료비 합 9,015원"
```

| 필요 데이터 | 공시 위치 | 실현 | 파서 |
|---|---|---|---|
| 원재료별 매입액·비중 | 「원재료 매입 현황」 | 🟢 | `auto_costmodel._parse_materials` (**있음**) |
| 원재료별 **매입수량** | 동 표(회사에 따라 수량열 존재) | 🟢/🟡 | **신규** `_parse_material_qty` |
| 원재료·제품 **단가** | 「주요 원재료 등의 가격변동추이」 | 🟢 | **신규** `_parse_price_trend` |
| **생산능력·생산실적·가동률** | 「생산 및 설비에 관한 사항」 | 🟢 | **신규** `_parse_production` |
| 제품 **판매단가** | 「주요 제품 등의 가격변동추이」 | 🟢 | **신규**(위와 공용) |
| 제품 중량·성분 | 식약처 식품안전나라·제품표시 | 🟡 | 수작업 KB(B급 업종) |
| 공학적 원단위 표준 | 업계 상수(고로 t당 철광석 1.5t 등) | 🟡 | 상수 테이블 |

> **가장 중요한 발견**: 「가격변동추이」표는 **회사가 실제로 산 단가**다. 국제시세가 아니다.
> → **회사 매입단가 vs 국제시세**의 괴리 자체가 강력한 신호(§11.4-③).

## 11.4 조작 탐지 — 물리 BOM이 재무제표를 감시하는 4가지 방법 ⭐ (사용자 요구의 핵심)

재무제표만 보면 못 잡는 것을, **물량 × 단가**를 곱해보면 잡힌다. 4개 크로스체크를 자동 실행한다.

### ① 이론 매출 vs 공시 매출 — **매출 부풀리기 탐지** 🟡
```
이론 매출 = Σ(제품 생산실적량 × 공시 판매단가)
괴리율   = (공시 매출 − 이론 매출) ÷ 이론 매출
```
- 공시 매출이 **물리적으로 만들 수 있는 양보다 크다** → 가공매출·순환거래 의심.
- 가동률 100% 초과 생산이 필요한 매출 → **물리 한계 위반 플래그**(가장 강력).

### ② 이론 원가 vs 공시 매출원가 — **원가 이연/재고 밀어내기 탐지** 🟡
```
이론 매출원가 = Σ(원단위 × 매입단가) × 생산량 + 가공비
```
- 공시 매출원가가 이론보다 **현저히 작다** → 원가를 재고자산으로 이연(재고 급증 동반 시 신호 강화).
- 이미 있는 §2.5 공손 플래그·`earnings_quality.py`와 **연계해 점수 합산**.

### ③ 회사 매입단가 vs 국제시세 — **특수관계자 거래·이전가격 탐지** 🟡
- 매입단가가 시세보다 **지속적으로 비싸다** → 계열사에 이익 이전 의심.
- **싸다** → 일시적 지원성 거래(이익 부풀리기) 의심. 3년 추세로 판정.

### ④ 원단위 3년 급변 — **수율·품질 조작 탐지** 🟢
- 원단위(투입량/생산량)는 설비가 안 바뀌면 **거의 안 변하는 공학 상수**.
- 갑자기 개선되면 → 실제 수율 개선인지, **투입량 과소계상**인지 검증 대상.

> 산출: **원가 신뢰도 스코어(0~100)** = 4개 체크 통과 여부 가중합 → 회사 상세 상단 배지.
> "재무제표 원가율 71%는 물리 BOM으로도 69~73% 범위 → **정합(신뢰)**" / "**괴리 12%p — 검증 필요**".

1. **B1 → B2**(뼈대·등급) → 치킨표 구조가 코드에 생김.
2. **B3 → B4**(공시 단가·물량 파서) → 여기서부터 **실측** 이 들어옴.
3. **B5 → B6**(원단위·조작탐지) ⭐ 사용자 요구의 핵심 가치.
4. **B7·B8**(KB 채우기) → **B9**(화면).

## 11.9 원칙 — 세분화하되 지어내지 않는다

- 모든 BOM 행에 **출처 배지**(🟢공시실측 / 🟡시세·추정 / ⚪내부데이터)와 **비고(가정)** 를 단다. 치킨표의 "비고"열이 정확히 이 역할.
- 치킨표처럼 **시나리오 2개 이상**(저가/고가 투입)을 병기해 단일 수치의 착시를 막는다.
- **제외 항목 명시**(치킨표의 "콜라 제외") — 무엇을 안 셌는지 보이는 게 신뢰의 조건.
- D급 업종은 **BOM을 만들지 않는다**. "물리 BOM 부적합" 배지 + 대체지표로 전환.
- 물리 BOM은 재무제표를 **대체하지 않고 감시**한다. 괴리가 나오면 어느 쪽이 틀렸는지 단정하지 말고 **양쪽과 가정을 함께 보여준다**.

# 12. 노무비(인건비) 레이어 (v3 신규) — "사람값"을 원가표에 넣는다

> 요구: *"노동자 가격은 하청도 있겠지만 회사마다 평균 연봉이 있잖아. 취업사이트에 잘 나와 있으니 그걸로 반영해라."*
> → **맞고, 더 좋은 소스가 있다. DART가 직원수·연간급여총액·1인평균급여를 이미 실측 공시한다(🟢).
> 취업사이트는 그 값의 교차검증 + 하청·미공시 단가 추정에 쓴다(🟡).**

## 12.1 왜 필요한가 — 치킨표의 결정적 구멍

치킨표는 **재료비만** 9,015원을 계산하고 "마진 8,985원"이라 적었다. 하지만 실제로는:

```
판매가 18,000원
 − 재료비   9,015원   ← 치킨표가 센 것
 − 인건비   ?????     ← 안 셈 (하루 50마리, 사장+알바 2명)
 − 임차료   ?????     ← 안 셈
 − 배달수수료·전기가스 ← 안 셈
 = 진짜 마진 (8,985원보다 훨씬 작다)
```

**인건비를 빼면 원가표는 항상 마진을 과대평가한다.** 대기업도 똑같다.
§11 물리 BOM은 재료비 축만 세운 상태이므로, **노무비를 두 번째 축으로 세워야 표가 닫힌다.**
(제조원가 3요소 = 재료비 + **노무비** + 경비. 지금 우리는 1/3만 정밀하다.)

## 12.2 데이터 소스 3층 — 사용자 제안(취업사이트)을 L2로 배치

| 층 | 소스 | 얻는 것 | 실현 | 현 코드 |
|:--:|---|---|:--:|---|
| **L1** | **DART `empSttus.json`** (사업보고서 「직원 등의 현황」) | 사업부문별 **직원수 · 연간급여총액 · 1인평균급여 · 근속연수 · 정규/기간제 구분** | 🟢 **실측** | `dart_profile._employees` **이미 구현** |
| **L2** | **취업사이트·연봉정보** (잡코리아·사람인·크레딧잡[국민연금 기반]·원티드) | 회사 평균연봉, **직무별**(생산직/사무직/연구직) 연봉, 비상장 하청업체 연봉 | 🟡 교차검증 | 신규 |
| **L3** | **고용노동부 고용형태공시제** (300인 이상 의무공시) | **소속 외 근로자 수 = 사내하도급(하청) 규모** | 🟢 | 신규 |
| L4 | DART 손익계산서 주석 · 제조원가명세서 | 급여·퇴직급여·복리후생비·**외주가공비** 계정 | 🟢 | 신규 파서 |

> **L1이 L2보다 우선인 이유**: DART는 법정 공시라 정확하고 **사업부문별로 쪼개져** 있어(생산/영업/연구) 제조노무비 분리가 된다.
> **L2(취업사이트)가 필요한 이유** — 사용자 지적대로 3가지에서 대체 불가:
> 1. **직무별 분해**: DART는 부문별이지 직무별 단가가 아님 → 생산직 시급 산정에 필요.
> 2. **하청업체 단가**: 하청은 비상장이라 DART 없음 → 취업사이트가 유일 소스.
> 3. **교차검증**: DART 1인평균급여 vs 시장 평균의 괴리 = **비정규직·하청 의존도 신호**(§12.5-⑥).
>
> ⚠️ 수집 원칙(§10.1과 동일): **사실(숫자)만** 취합, 원문·리뷰 텍스트 복제 금지, 개인용·레이트리밋 준수, 출처 링크 병기.

## 12.3 계산식 — 제품 1단위 노무비 만들기

```
[식4] 총인건비        L      = Σ(부문 직원수 × 부문 1인평균급여) + 퇴직급여·복리후생  🟢 L1
[식5] 제조노무비      L_mfg  = L × (생산부문 인원 ÷ 전체 인원)                        🟢 L1(부문별 rows)
[식6] 단위 노무비     l_unit = L_mfg ÷ 생산수량                                       🟡 (§11.3 식2의 생산실적 재사용)
[식7] 외주(하청) 노무비 L_out = 소속외 근로자수 × 하청 추정연봉                        🟡 L3 × L2
                             ※ 검증: 제조원가명세서 「외주가공비」 계정과 대조 🟢 L4
[식8] 완전 노무비     L_all  = L_mfg + L_out   ← "하청까지 포함한 진짜 사람값"
```

> **핵심 포인트**: 대기업 원가율이 낮아 보이는 흔한 이유가 **인건비를 사외(하청)로 밀어낸 것**이다.
> L_out을 안 세면 "인건비 효율 좋은 회사"로 오독한다. 식7·식8이 이걸 교정한다.

## 12.5 조작 탐지 확장 — §11.4에 ⑤⑥⑦ 추가 ⭐

인건비는 **분식의 단골 통로**다. 물량(인원)은 공시되므로 여기서도 크로스체크가 가능하다.

### ⑤ **인건비 자본화 탐지** (가장 강력) 🟢
```
인원은 유지·증가  +  비용화 인건비는 급감  +  무형자산(개발비) 급증  →  🚩
```
인건비를 개발비(무형자산)로 돌려 당기 이익을 부풀리는 전형적 수법(바이오·게임·IT·조선).
**인원은 못 숨긴다** → 인당 비용화 인건비의 급락이 곧 신호. `earnings_quality.py`와 점수 연계.

### ⑥ **DART 평균급여 vs 취업사이트 평균 괴리** 🟡 ← 사용자 제안의 직접 활용
- DART 평균급여 **≪** 시장 평균 → 저임금·기간제 비중 높음 or **하청 의존**(원가가 사외 이전).
  → L3(소속외 근로자 수)와 **함께 봐야** 진짜 인건비가 보인다.
- DART 평균급여 **≫** 시장 평균 → 임원·고연봉 편중, 또는 인원 축소 후 급여총액 유지(구조조정 진행형).

### ⑦ **인당 지표 급변** 🟢
- 인당매출이 3년새 급증했는데 인원·설비 증가가 없다 → **매출 부풀리기 방증**(§11.4-①과 교차).
- 인건비/매출 비율이 업종 평균 대비 이상치 → 외주 전가 or 계상 누락.

> §11.4의 원가 신뢰도 스코어(0~100)에 ⑤⑥⑦을 항목으로 편입한다.

# 13. 재무제표 3종 감사 (v3 신규·구현완료) — 표끼리 물려 있는 걸 이용한다

> 요구: *"재무상태표·현금흐름표·손익계산서가 잘 되어 있는지 확인도 해주고, 큰 회사일수록 조작은 허다할 것 같다. 그걸 찾아내는 게 우리 임무다."*

## 13.1 원리 — 손익은 다듬기 쉽고, 현금은 어렵다

재무제표 3종은 **서로 물려(articulate) 있다.** 한 곳을 손보면 다른 곳이 어긋난다.
그래서 "표가 다 있는가"(커버리지)와 "표끼리 말이 맞는가"(정합성)를 같이 본다.

| 코드 | 검증 | 잡아내는 것 |
|---|---|---|
| **A1** | 자산총계 = 부채총계 + 자본총계 | 재무상태표 자체 오류·적재 오류 |
| **A2** | 매출 − 매출원가 − 판관비 = 영업이익 | 손익 구조 오류 |
| **A3** | 영업CF + 투자CF + 재무CF + 환율효과 = 기말현금 − 기초현금 | 현금흐름표 조작·누락 |
| **D1** | **발생액 비율** = (순이익 − 영업CF) ÷ 자산 | **이익은 나는데 현금이 없다** ⭐ |
| **D2** | 순이익 > 0 인데 영업CF < 0 | 장부이익과 현금의 방향이 반대 — 분식 대표 신호 |
| **D3** | 매출채권 증가율 ≫ 매출 증가율 | 밀어내기·가공매출 |
| **D4** | 재고 증가율 ≫ 매출원가 증가율 | 원가 이연(§11.4-②와 동일 논리) |
| **D5** | 자본총계 < 0 | 자본잠식 |

**부호가 중요하다**: 발생액이 **(+)** 면 위험(이익>현금), **(−)** 면 오히려 보수적(현금>이익).
감가상각이 큰 장치산업은 (−)가 정상이라 절대값으로 보면 멀쩡한 회사가 경고로 찍힌다.

## 13.2 구현에서 잡은 **오탐의 3대 원인** (실측하며 교정)

정확도가 생명이다. 삼성전자를 "분식 의심"으로 찍는 탐지기는 없느니만 못하다.

| 원인 | 증상 | 교정 |
|---|---|---|
| **연결(CFS)·별도(OFS) 혼재** | 자산은 연결, 자본은 별도가 잡혀 항등식 −2% 깨짐 | **연도마다 하나의 기준으로 통일**(연결 우선), 화면에 연결/별도 표기 |
| **동명 계정 혼입** | '자본총계'가 **자본변동표에도** 있어 기초잔액이 잡힘(S-Oil −34%) | 계정마다 **찾을 표(sj_div)를 고정** |

## 14.1 발견 — 사업보고서 원문에 주석·감사보고서가 통째로 있다

`document.xml` API로 받은 zip 안에는 본문(사업의 내용)뿐 아니라 **재무제표 주석 전문**과
**감사보고서 전문**이 별도 멤버 파일(연결/별도)로 들어 있다. 즉 우리가 추정하던 값의 **원본**이 있다.

| 지금까지 | 원문에서 실측 |
|---|---|
| `material_ratio_of_cogs = 0.80` **가정** | 「비용의 성격별 분류」 주석의 **원재료 사용액** |
| 노무비 = 잔차에 뭉뚱그림 | 같은 주석의 **종업원급여** |
| 감가상각 = 가공비에 섞임 | 같은 주석의 **감가상각비·무형자산상각비** |
| 회계 신뢰도 = 우리 계산만 | **감사의견 · 핵심감사사항(KAM) · 계속기업 불확실성** |

| KAM 제목에 감사절차 문장 혼입 | 제목이 앞 문단과 같은 셀에 있는 경우 | 직전 **독립 텍스트 노드** 우선 + 절차 서술어 필터, 실패 시 **버리고 건수만 보고**(지어내지 않기) |

### 새 검증 **V1 — 생산량 증가율 vs 매출 증가율** (§11.4-① 실행 가능한 형태)

단위 환산이 필요 없는 **증가율 비교**라 업종을 안 가린다. 물량이 제자리인데 매출만 뛰면 의심.

|---|--:|--:|---|
| 농심 | +12.2% | +0.8% | 정상(물량이 더 늚) |
| POSCO홀딩스 | −4.0% | −5.8% | 정상 |
| S-Oil | +1.2% | +2.5% | 정상(단가 반영) |

### 파싱에서 잡은 함정 (또 실측이 코드를 고쳤다)

| 증상 | 원인 | 교정 |
|---|---|---|
| 농심 생산량 증가율 **+1875%** | 기수 헤더 아래 **수량/금액 2단 서브헤더** → 한 해는 수량, 다른 해는 금액이 잡힘 | 서브헤더 감지 후 **수량 열만** 사용, 정렬 불가하면 **표를 버림** |
| 원재료 품목명이 "수입" | 품목 칸이 병합되어 **구분 열(수입/국내)** 이 이름으로 | 구분값 사전 제외 + 앞 3칸 결합 |
| 삼성전자 "원재료 단가"가 생산실적과 동일 | 가격변동추이 절에 표가 없어 **다음 절의 표**를 집어옴 | 제목-표 거리 제한(2,500자) + 생산실적과 중복되면 **블록 폐기** |
| — | 값이 1년에 ±200% 넘게 변하는 행 | 파싱오류로 보고 **제외 + 제외 건수 표기** |

> POSCO는 「가격변동추이는 **정보유출 우려 등을 감안하여 미표기**」라고 공시한다 —
> 안 적는 회사가 많다. 없으면 없다고 표시하고 지어내지 않는다.
